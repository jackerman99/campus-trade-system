\# V1 范围定义（scope-v1）



\## 1. 项目名称



基于 Spring Cloud 的校园二手交易微服务系统



\## 2. 项目目标



本项目旨在实现一个基于 Spring Cloud 的校园二手交易微服务系统，完成用户注册登录、商品发布、商品浏览、商品搜索、我的商品管理、管理员审核等核心功能，并通过 Nacos、Gateway、JWT、MySQL、Jenkins 等技术完成微服务架构搭建、服务治理与部署上线，满足课程大作业对微服务开发、CI/CD、页面数量、代码规模和系统上线的要求。



\## 3. 本次实现范围



本次 V1 版本实现以下功能：



1\. 用户注册

2\. 用户登录

3\. JWT 鉴权与登录态校验

4\. 商品列表展示

5\. 商品详情查看

6\. 商品发布（支持商品图片上传）

7\. 商品搜索

8\. 我的商品管理

9\. 管理员审核商品

10\. 微服务注册与发现

11\. 网关统一路由转发

12\. Jenkins 自动构建

13\. 系统部署上线并可访问



\## 4. 本次不实现内容



为了保证项目按时完成并满足课程硬性要求，本次 V1 版本不实现以下扩展功能：



1\. AI 描述润色

2\. 智能定价建议

3\. 智能运营报表

4\. MCP 或类似智能服务集成

5\. 用户留言/聊天系统

6\. 复杂信誉分动态管理



\## 5. 系统角色



本系统在 V1 版本中包含以下两类角色：



\### 5.1 普通用户（USER）

\- 注册账号

\- 登录系统

\- 浏览商品

\- 搜索商品

\- 发布商品

\- 查看我的商品



\### 5.2 管理员（ADMIN）

\- 登录系统

\- 查看待审核商品

\- 审核通过商品

\- 驳回商品



\## 6. 微服务拆分



本项目 V1 版本采用如下微服务划分：



1\. gateway-service

&#x20;  - 统一路由转发

&#x20;  - 跨域处理

&#x20;  - 统一鉴权入口



2\. user-service

&#x20;  - 用户注册

&#x20;  - 用户登录

&#x20;  - 当前用户信息查询



3\. item-service

&#x20;  - 商品发布

&#x20;  - 商品列表

&#x20;  - 商品详情

&#x20;  - 商品搜索

&#x20;  - 我的商品管理

&#x20;  - 图片上传



4\. admin-service

&#x20;  - 待审核商品列表

&#x20;  - 商品审核通过

&#x20;  - 商品审核驳回



5\. common-module

&#x20;  - 公共返回结构

&#x20;  - 公共异常

&#x20;  - 枚举定义

&#x20;  - 通用工具类



\## 7. 核心业务闭环



本系统 V1 版本的核心业务闭环为：



用户注册/登录 -> 浏览商品 -> 搜索商品 -> 发布商品 -> 管理员审核商品 -> 审核通过后商品在首页可见



该业务闭环覆盖了用户中心、商品中心、搜索能力、后台审核能力以及前后端联调的核心路径，是本次项目演示和验收的主线流程。



\## 8. 页面范围



本项目 V1 版本前端页面范围如下：



1\. 登录页

2\. 注册页

3\. 首页商品列表页

4\. 商品详情页

5\. 发布商品页

6\. 我的商品页

7\. 管理员审核页

\- JWT

\### 前端

\### 工具与部署

\- Git / GitHub

\- Jenkins

\- Linux 服务器

\- Nginx



\## 10. 状态枚举范围



\### 10.1 用户角色

\- USER：普通用户

\- ADMIN：管理员



\### 10.2 用户状态

\- ACTIVE：正常

\- DISABLED：禁用



\### 10.3 商品状态

\- ON\_SALE：已上架

\- SOLD：已卖出

\- OFFLINE：已下架



\### 10.4 审核状态

\- PENDING：待审核

\- APPROVED：审核通过

\- REJECTED：审核驳回



\## 11. 开发边界说明



本文件用于冻结项目 V1 版本的实现范围。后续数据库设计、接口设计、后端开发、前端开发、测试与部署均以本文件为准。



若后续需要增加扩展功能，应在 V1 核心功能全部完成并通过联调后，再单独评估是否纳入二期版本，不在本次课程大作业的必做范围内。
